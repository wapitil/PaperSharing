# 课题组论文阅读沉淀工具

## 一句话概念

做一个面向课题组传承的论文阅读知识沉淀工具：不替代 Zotero、Obsidian、飞书等已
有阅读/笔记工具，而是把不同成员已有的论文笔记汇聚到同一篇论文下面，整理成新人
容易理解的阅读入口。

## 背景问题

课题组里经常有人读过某篇论文、做过组会汇报、踩过实验坑，但这些理解通常散落在
PPT、飞书文档、Obsidian 笔记、Zotero 批注、聊天记录里。

结果是：

- 前人的阅读经验难以复用
- 新生读论文时仍然从零开始
- 导师和师兄师姐的判断没有长期沉淀
- 多个人对同一篇论文的理解无法被统一浏览
- 文献管理工具只管理“论文”，没有很好管理“组内怎么看这篇论文”

## 核心目标

让论文阅读从一次性个人行为，变成可积累、可传承的组内知识资产。

系统重点不是读 PDF，而是回答新人最关心的问题：

- 这篇论文为什么要读？
- 应该重点看哪里？
- 哪些部分可以跳过？
- 前人怎么看这篇论文？
- 这篇论文有什么坑或容易误解的地方？
- 它和本课题组方向有什么关系？
- 现在是否仍然值得读？

## 核心原则

1. 不替代现有工具
    用户可以继续用 Zotero 管文献，用 Obsidian 写笔记，用飞书做组会记录。

2. 低成本上传
    分享者不需要重新写完整综述，只需要粘贴文本、导入 Markdown，或上传已有笔记。

3. 统一输出结构
    不管笔记来自哪里，最后都汇聚到同一篇论文页面下面。

4. 新人优先
    新人默认看到整理后的阅读入口，而不是一堆混乱的原始笔记。

5. 保留个人观点
    多个分享者的理解不强行合并，个人观点、组内共识、争议和原始笔记分层展示。

## 最小产品形态

第一版只需要三个核心对象：

Paper：论文
Note：某位成员的笔记或观点
Curated View：给新人看的整理入口

关系是：

一篇 Paper
包含多个 Note
有一个 Curated View

## 分享者流程

分享者可以通过最小步骤上传内容：

1. 搜索或新建一篇论文
2. 粘贴笔记 / 导入 Markdown / 上传已有文本
3. 填写作者身份
4. 选择是否给新人可见
5. 提交


如果系统无法准确分类，就先保留为原始笔记。

## 新人阅读界面

新人打开一篇论文时，默认看到分层内容：

论文标题
推荐程度 / 标签 / 适合人群

1. 新人先看
    - 为什么读这篇
    - 建议重点看什么
    - 可以跳过什么
    - 读前需要知道什么

2. 组内共识
    - 当前大家基本认可的判断
    - 这篇是否仍然推荐阅读
    - 和本组方向的关系

3. 高价值提醒
    - 实验坑
    - 常见误解
    - 已过时之处
    - 后续替代论文

4. 分享者观点
    - 每个人的观点卡片
    - 默认显示摘要
    - 可展开完整笔记

5. 原始笔记
    - 保留上传者的完整内容
    - 需要时再展开

## 多人笔记如何避免混乱

系统不把所有人的笔记直接平铺，而是分层展示：

- 整理入口：给新人快速开始
- 组内共识：导师或管理员整理
- 高价值提醒：从多人笔记中提取
- 个人观点：按分享者独立展示
- 原始笔记：作为可追溯材料保留

个人观点不强行合并。
如果出现不同意见，可以形成“争议区”：

争议点：这篇论文的实验是否公平？

观点 A：baseline 偏弱
支持者：A、B

观点 B：在当时条件下合理
支持者：C

当前状态：未形成共识

## 可追溯性设计

第一版不做完整 PDF 阅读器，也不追求像 Zotero 一样的精确批注定位。

采用三档追溯方式：

精确定位：来自 PDF/Zotero 批注，带页码或原文位置
弱定位：笔记中包含原文摘录，系统尝试匹配到论文文本
无定位：整体观点，不强制绑定原文

重点是让新人知道：

- 这是谁的观点
- 是否有原文依据
- 是否是组内共识
- 是否只是个人判断

## 第一版不做的事情

为了保持最小实现，第一版暂时不做：

- PDF 阅读器
- Zotero 插件
- 飞书 API 同步
- 实时多人协作
- 完整批注定位
- 复杂评分系统
- 自动生成完美综述

## 第一版要验证的问题

MVP 主要验证三件事：

1. 分享者是否愿意把已有笔记粘贴或导入进来？
2. 新人是否能通过这个页面更快开始读论文？
3. 多个人对同一篇论文的理解放在一起后，是否仍然清楚、不混乱？

## 最终愿景

长期来看，它可以发展成课题组的论文阅读记忆库。

它不是简单的文献管理器，而是记录：

- 谁读过这篇论文
- 他们当时怎么理解
- 后来是否改变看法
- 这篇论文在组内研究方向中的位置
- 哪些经验可以传给后来的学生

一句话总结：

> 这是一个不替代现有阅读工具的论文笔记汇聚与传承系统，让课题组成员把散落在
> Zotero、Obsidian、飞书等工具中的阅读理解沉淀到同一篇论文下，并以新人友好的方
> 式展示出来。
