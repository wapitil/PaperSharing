# 更新日志

## 2026-06-17

### 新增

- 新增独立阅读端 `reader.html` 和 `reader.js`。
- 新增独立管理端 `admin.html` 和 `admin.js`。
- 阅读端支持从本地存储读取分享端保存的阅读经验。
- 阅读端支持按论文聚合阅读经验。
- 阅读端支持论文搜索，可搜索标题、作者、年份和链接。
- 管理端支持查看全部本地记录。
- 管理端支持展开记录后直接用表单查看和编辑已有阅读经验。
- 管理端移除单条记录详情中的原始 JSON 展示。
- 管理端支持删除单条经验。
- 管理端支持清空全部本地数据。
- 管理端支持导出和导入 JSON。
- 新增 Node 后端 `server.js`，提供静态文件服务和 `/api/notes` 数据接口。
- 新增最小登录身份：前端支持注册/登录，后端使用 HttpOnly cookie session 识别操作者。
- 新增独立登录页 `login.html`，顶部身份区只保留登录入口、当前身份和退出。
- 新增 TipTap 富文本编辑器，正文、阅读点和个人看法支持粘贴图片并随正文排版。
- 新增 Vite 多页面构建，Node 后端托管 `dist/` 产物。
- 管理端只读取、修改和删除当前身份发表的阅读经验。
- 阅读端保持只读，删除和清空操作后续移到管理端。

### 调整

- 分享端顶部入口从“查看阅读界面”调整为“查看阅读库”。
- 阅读端顶部入口从“去分享”调整为“添加阅读经验”。
- 阅读端仅保留“按分享者查看”的阅读方式。
- 阅读端分享者经验卡片改为可折叠展开。
- 阅读端论文列表按最近上传时间排序。
- 同一论文下的分享者经验按上传时间倒序展示。
- 分享端保存时自动绑定当前身份，不再把分享者名称作为自由填写字段。
- 默认首页从编辑页调整为公开阅读库，未登录用户也可以浏览分享内容。
- 添加阅读经验页在发表时才要求登录，并在跳转登录前临时保存草稿。
- 阅读经验正文从纯文本扩展为 ProseMirror JSON，同时保留旧纯文本字段兼容历史数据。
- 顶部同级操作统一为相同字体、字号和按钮高度。
- 前端资源整理到 `assets/css` 和 `assets/js`，根目录保留入口页面和项目文档。
- 管理端展开入口文案改为“编辑”。
- 前端数据读写改为优先使用后端 API，后端不可用时回退到 `localStorage`。

### 校验

- 分享端保存时新增内容校验：至少需要填写“为什么选择这一篇文章”、一个阅读点或“我的看法”之一。
- 图片粘贴限制为单张 3MB，单条阅读经验限制为约 10MB。
- 新增零依赖冒烟测试 `tests/smoke-test.mjs`，检查页面引用、脚本语法、核心功能入口和本地服务响应。
- 新增 `package.json`，提供本地服务和测试命令。
- 新增后端 API 测试 `npm run test:backend`，验证 `/api/health`、`/api/notes` 和静态页面服务。
- 新增可选网络测试 `npm run test:network`，验证 DOI/arXiv 论文识别依赖的外部接口。

### 说明

- 当前数据默认保存在 `data/papersharing.sqlite` 中。
- 当前版本用于验证分享端、阅读端、管理端、最小登录和基础数据保存流程。
