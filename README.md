# PaperSharing

面向课题组的论文阅读经验沉淀工具原型。

这个项目不是 Zotero、Obsidian 或飞书文档的替代品。它关注的是另一件事：把组内成员读论文时留下的判断、卡点、误解、复现实验注意事项等经验，沉淀到同一篇论文下面，方便后来的人快速了解“组里是怎么读这篇论文的”。

## 当前状态

这是一个纯前端静态原型，不需要安装依赖，也没有后端服务。

当前主要包含两个页面：

- `index.html`：分享端，用于录入某位成员的一次论文阅读经验。
- `reader.html`：阅读端，用于按论文查看已经沉淀的多人阅读经验。
- `admin.html`：管理端，用于维护本地原型数据。

数据暂时保存在浏览器的 `localStorage` 中，用于验证最小产品流程。

## 核心功能

### 分享端

分享者可以完成以下操作：

1. 输入 DOI、arXiv ID 或论文链接。
2. 自动识别论文标题、作者、年份和链接。
3. 手动修正识别失败或不完整的论文信息。
4. 手动填写分享者名称。
5. 选择整体判断：
   - 很有用
   - 只用到一部分
   - 暂时没帮上忙
   - 还没判断清楚
6. 记录“为什么选择这一篇文章”。
7. 按类型添加多条阅读点：
   - 我卡住的地方
   - 我觉得重要的地方
   - 我不确定的判断
   - 容易误解的点
   - 和自己课题有关的点
   - 复现/实验注意事项
8. 保存后生成上传时间，并写入本地存储。

### 阅读端

阅读端用于验证多人经验聚合后的浏览方式：

1. 按论文分组展示阅读经验。
2. 显示每篇论文下已有多少条经验。
3. 支持按标题、作者、年份和链接搜索论文。
4. 聚合显示整体判断数量。
5. 按分享者展示完整经验卡片。
6. 普通阅读端保持只读，不提供删除和清空操作。

### 管理端

管理端用于维护本地原型数据：

1. 查看全部本地记录。
2. 删除单条经验。
3. 清空全部本地数据。
4. 导出 JSON 备份。
5. 导入 JSON 恢复或迁移数据。

## 本地运行

直接用浏览器打开文件即可：

```bash
open index.html
```

或打开阅读端：

```bash
open reader.html
```

如果浏览器限制本地文件请求，建议启动一个简单静态服务器：

```bash
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000/index.html
http://localhost:8000/reader.html
```

## 设计原则

- 不替代现有论文管理和笔记工具。
- 先降低分享成本，再追求自动化。
- 允许个人经验不完整、不标准、不权威。
- 多人经验不强行合并，先保留来源和个人判断。
- 阅读端优先保证清楚，不把所有笔记直接平铺。

## 后续方向

可能的下一步包括：

- 接入真实账户系统，替代手动填写分享者名称。
- 增加后端存储，替代 `localStorage`。
- 支持按论文搜索和筛选。
- 支持从 Markdown 或已有笔记导入。
- 为导师或管理员增加“组内共识”整理能力。
- 将分享端和阅读端拆成更正式的应用路由。

## 说明

当前版本重点验证两个问题：

1. 分享者是否愿意用低成本方式留下阅读经验。
2. 多个人对同一篇论文的经验放在一起后，阅读端是否仍然清楚。

因此，这个阶段有意避免 PDF 阅读器、复杂权限、实时协作、Zotero 插件和自动综述生成。
