# PaperSharing

面向课题组的论文阅读经验沉淀工具原型。

这个项目不是 Zotero、Obsidian 或飞书文档的替代品。它关注的是另一件事：把组内成员读论文时留下的判断、卡点、误解、复现实验注意事项等经验，沉淀到同一篇论文下面，方便后来的人快速了解“组里是怎么读这篇论文的”。

## 当前状态

这是一个轻量级 Web 原型，前端使用 Vite 管理原生 HTML/CSS/JavaScript 模块，后端使用 Node.js 内置模块实现。

当前主要包含四个页面：

- `reader.html`：公开阅读库，用于按论文查看已经沉淀的多人阅读经验，也是默认首页。
- `index.html`：添加阅读经验页，用于录入某位成员的一次论文阅读经验。
- `admin.html`：管理端，用于维护自己发表的阅读经验。
- `login.html`：登录/注册页，登录后默认回到阅读库，带 `next` 参数时回到来源页面。

数据默认保存在本地 SQLite 数据库 `data/papersharing.sqlite` 中。用户通过邮箱、显示名和密码注册或登录，后端使用 HttpOnly cookie session 识别当前用户。阅读经验正文使用 TipTap/ProseMirror JSON 保存，支持粘贴图片并随正文排版显示。

## 核心功能

### 添加阅读经验页

发表者可以完成以下操作：

1. 输入 DOI、arXiv ID 或论文链接。
2. 自动识别论文标题、作者、年份和链接。
3. 手动修正识别失败或不完整的论文信息。
4. 不登录也可以先编辑草稿，点“发表”时再登录。
5. 选择整体判断：
   - 很有用
   - 只用到一部分
   - 暂时没帮上忙
   - 还没判断清楚
6. 记录“为什么选择这一篇文章”，可直接粘贴截图。
7. 按类型添加多条阅读点：
   - 我卡住的地方
   - 我觉得重要的地方
   - 我不确定的判断
   - 容易误解的点
   - 和自己课题有关的点
   - 复现/实验注意事项
8. 发表后生成上传时间，并写入后端数据库。

图片第一版以 base64 保存在结构化正文里：单张粘贴图片限制为 3MB，单条阅读经验限制为约 10MB。后续可以把图片 `src` 从 base64 平滑替换为上传后的文件 URL。

### 公开阅读库

公开阅读库用于验证多人经验聚合后的浏览方式，未登录用户也可以访问：

1. 按论文分组展示阅读经验。
2. 显示每篇论文下已有多少条经验。
3. 支持按标题、作者、年份和链接搜索论文。
4. 聚合显示整体判断数量。
5. 按分享者展示完整经验卡片。
6. 阅读库保持只读，不提供删除和清空操作。

### 管理端

管理端用于维护当前登录用户发表的阅读经验：

1. 查看自己发表的记录。
2. 展开单条记录后直接用表单查看和编辑自己发表的经验。
3. 删除单条经验。
4. 清空自己发表的全部数据。
5. 导出 JSON 备份。
6. 导入 JSON 恢复或迁移数据。

## 本地运行

建议通过内置后端启动：

```bash
npm run serve
```

`npm run serve` 会先执行 `npm run build`，再由 Node 后端托管 `dist/` 中的页面和静态资源。

然后访问：

```text
http://localhost:8000/index.html
http://localhost:8000/reader.html
http://localhost:8000/admin.html
http://localhost:8000/login.html
```

## 测试

当前原型没有引入测试框架，提供了一个零依赖的冒烟测试脚本，用于检查页面引用、脚本语法、Vite 构建和核心功能入口：

```bash
node tests/smoke-test.mjs
```

也可以使用 npm 脚本：

```bash
npm test
```

`npm test` 会先构建前端。如果本地 `8000` 端口已经启动静态服务器，脚本还会顺带检查四个页面是否能通过 HTTP 正常访问。

后端 API 可以单独测试：

```bash
npm run test:backend
```

论文识别依赖外部接口，必要时可以单独运行网络检查：

```bash
npm run test:network
```

## 设计原则

- 不替代现有论文管理和笔记工具。
- 先降低分享成本，再追求自动化。
- 允许个人经验不完整、不标准、不权威。
- 多人经验不强行合并，先保留来源和个人判断。
- 阅读端优先保证清楚，不把所有笔记直接平铺。

## 后续方向

可能的下一步包括：

- 接入真实账户系统，替代当前最小身份模型。
- 支持图片上传和插入。
- 支持从 Markdown 或已有笔记导入。
- 为导师或管理员增加“组内共识”整理能力。
- 将分享端和阅读端拆成更正式的应用路由。

## 说明

当前版本重点验证两个问题：

1. 分享者是否愿意用低成本方式留下阅读经验。
2. 多个人对同一篇论文的经验放在一起后，阅读端是否仍然清楚。

因此，这个阶段有意避免 PDF 阅读器、复杂权限、实时协作、Zotero 插件和自动综述生成。

当前版本使用最小登录模型：后端保存用户、密码哈希和 session，浏览器只持有 HttpOnly session cookie。它已经能防止前端随意伪造身份，但仍然是原型级实现，尚未包含注册审核、登录限流、CSRF 防护和更完整的账户管理。
